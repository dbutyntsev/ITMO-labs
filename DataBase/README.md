# 🐘 Relational Databases (PostgreSQL)

Этот раздел посвящен изучению реляционных баз данных, проектированию оптимальных схем данных, написанию комплексных аналитических SQL-запросов и оптимизации производительности СУБД (PostgreSQL).

---

### 📌 Структура курса и этапы разработки:

#### 🔹 Модуль 1: Моделирование и реализация доменной модели (Лабораторная работа №1)
- Анализ текстового сценария предметной области, выделение сущностей, их атрибутов и типов связей.
- Разработка инфологической (ER-диаграмма) и даталогической моделей данных с учетом ограничений целостности (Primary/Foreign Keys, Check, Unique, Not Null).
- Создание таблиц и наполнение их тестовыми данными с помощью DDL/DML.

#### 🔹 Модуль 2: Реляционная алгебра и сложные SQL-запросы (Лабораторная работа №2)
- Построение комплексных выборок с использованием различных типов соединений (`INNER JOIN`, `RIGHT JOIN`), агрегатных функций, группировок (`GROUP BY`, `HAVING`) и сложных подзапросов (включая предикат `EXISTS`).

#### 🔹 Модуль 3: Нормализация отношений и процедурный SQL (Лабораторная работа №3)
- Анализ функциональных зависимостей разработанной схемы, приведение таблиц к третьей нормальной форме (3NF) и форме Бойса-Кодда (BCNF). Изучение компромиссов контролируемой денормализации.
- Написание хранимых процедурных функций и создание триггеров для обеспечения автоматизации бизнес-логики.

#### 🔹 Модуль 4: План выполнения запросов и индексация (Лабораторная работа №4)
- Изучение планов выполнения запросов (Query Execution Plans) в условиях отсутствия индексов на больших таблицах СУБД.
- Проектирование и внедрение эффективных индексов (B-Tree) под конкретные фильтры и условия соединений.
- Инструментальный анализ изменения стоимости и времени выполнения операций с помощью утилиты `EXPLAIN ANALYZE`.
